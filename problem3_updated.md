下面给你一版**可以直接写进论文**的“问题3求解与结果分析”。我把它做成了一个比问题1、2更“算法化”的框架：**对手风格识别 + 半马尔可夫决策过程（SMDP）+ 蒙特卡洛树搜索（MCTS）+ 时间扩展网络动态规划**。这样既能体现数学建模深度，也能把“先后采取哪些攻击动作与防守动作”真正落到可计算的策略序列上。题目要求的是：**在不考虑故障的前提下，对5分钟单场竞技赛建立单人比赛策略优化模型，并给出最佳作战决策方案**。

## 一、问题3的核心思路

问题1已经给出了13种攻击动作的动力学评分框架：攻击收益由末端速度、等效质量、命中概率等决定，稳定风险由质心偏移、角速度、单腿支撑比例等决定，并通过综合评分衡量动作优劣。

问题2已经把13种攻击动作与22种防守动作做了匹配，得到“主防动作—辅助稳定动作—兜底动作”的闭环防守方案，并给出了不同攻击类型下的最佳防守动作。

因此，问题3不应再孤立地比较某一个动作强不强，而应转化为：

[
\text{在比赛动态状态 }s_t\text{ 下，选择动作 }a_t\in(\text{攻击集}\cup\text{防守集}),
]
使得在整个比赛时域内的**累计期望收益最大**，也就是最终获胜概率最大。

---

## 二、建模假设

为使模型可求解，作如下假设：

1. 单场比赛不考虑机器人故障、人工复位、战术暂停和紧急维修，仅保留正常攻防博弈。题目中这些资源约束属于问题4处理范围。

2. 机器人动作仍沿用问题1的参数化结果，即每个攻击动作都有
   [
   (A_i,S_i,U_i,q_i,e_i,T_i)
   ]
   等指标，其中 (A_i) 表示攻击收益，(S_i) 表示稳定性，(q_i) 表示衔接性，(e_i) 表示暴露风险。

3. 防守动作仍沿用问题2的匹配结果，即面对不同攻击原型，存在最优主防动作 (j_i^*)，并可补充辅助动作与兜底动作形成闭环。

4. 比赛是一个**离散决策过程**：每完成一次动作或一次攻防交换，就进入下一决策时刻。由于不同动作时长不同，使用**半马尔可夫决策过程**比普通马尔可夫决策过程更合适。动作时长在问题1中已按短、中、长动作划分。

## 三、状态空间与动作空间

### 1. 状态变量

定义比赛时刻 (t) 的状态为

[
s_t=(\tau_t,d_t,g_t,b_t^s,b_t^o,\hat z_t),
]

其中：

* (\tau_t)：剩余比赛时间；
* (d_t\in{\text{远距离},\text{中距离},\text{近距离}})：双方距离状态；
* (g_t\in{\text{我方先手},\text{相持},\text{对方先手}})：节奏主导权；
* (b_t^s\in{\text{稳定},\text{扰动},\text{失衡}})：我方平衡状态；
* (b_t^o\in{\text{稳定},\text{扰动},\text{失衡}})：对方平衡状态；
* (\hat z_t\in{1,\dots,6})：由近期观测动作序列经 Viterbi 识别得到的对手当前攻击风格类别。

这里的6类攻击风格直接沿用问题2对攻击原型的划分：直线快攻型、弧线摆动型、旋转腿击型、低位破坏型、贴身压迫型、特殊场景型。

### 2. 动作空间

定义动作集合

[
\mathcal A=\mathcal A_{\text{atk}}\cup \mathcal A_{\text{def}}.
]

其中：

* 攻击动作共13种，来自题目附录一；
* 防守动作共22种，来自题目附录二；

但在问题3中，并不是13种攻击、22种防守都要高频使用。依据你上传的动作评分表 [13个动作评分结果.csv](sandbox:/mnt/data/13个动作评分结果.csv)，主动攻击优先级可分为三层：

* **主攻动作池**：冲撞、直拳、前踢、组合拳；
* **辅助动作池**：勾拳、摆拳、侧踢、膝撞；
* **条件触发动作池**：拳腿组合、回旋踢、低扫腿、五连踢、倒地反击。

其中，综合得分靠前的是冲撞、直拳、前踢、组合拳；而回旋踢、五连踢、倒地反击综合得分偏低，更适合作为特定场景动作，而不适合作为常规主攻链。

## 四、对手攻击原型转移建模与节奏识别

### 1. 攻击原型转移建模

为了让策略不是“固定出招表”，而是能根据对手变化自适应，先建立对手攻击原型的马尔可夫转移模型。设隐藏状态 \(z_t\) 表示时刻 \(t\) 对手真实所处的攻击原型，取值为问题2划分的六类攻击风格：直线快攻型、弧线摆动型、旋转腿击型、低位破坏型、贴身压迫型和特殊场景型。则其状态转移满足一阶马尔可夫性：

[
P(z_t=k\mid z_{1:t-1})=P(z_t=k\mid z_{t-1})=a_{z_{t-1}k},
]

其中 \(a_{ij}\) 表示攻击原型 \(i\) 向攻击原型 \(j\) 的转移概率。该模型用于描述对手在比赛过程中的节奏延续性与切换规律，例如直线快攻通常会在短时间内保持较高惯性，而贴身压迫型在近距离建立后也更容易持续若干拍。

### 2. 攻击节奏识别与状态更新

仅有原型转移矩阵还不够，因为比赛中并不能直接观测到对手当前属于哪一类攻击原型，实际可获得的信息通常只是最近若干拍的外显动作序列，如直拳、前踢、摆拳、侧踢、冲撞等。因此，本文在马尔可夫链基础上进一步引入**攻击节奏识别算法**：依据近期已观测动作序列，实时估计当前最可能的攻击原型状态，再将识别结果作为后续决策层的状态输入。

设观测序列为

[
O_{1:t}=(o_1,o_2,\dots,o_t),
]

其中 \(o_t\) 表示第 \(t\) 次观测到的对手攻击动作。给定当前攻击原型后，观测动作满足条件独立性：

[
P(o_t\mid z_{1:t},o_{1:t-1})=P(o_t\mid z_t)=b_{z_t}(o_t),
]

其中 \(b_{z_t}(o_t)\) 表示在攻击原型 \(z_t\) 下观测到动作 \(o_t\) 的发射概率。于是，对手节奏识别问题可转化为给定观测序列 \(O_{1:t}\) 后，求解最大后验路径：

[
Z_{1:t}^*=\arg\max_{Z_{1:t}}P(Z_{1:t}\mid O_{1:t})
=\arg\max_{z_1,\dots,z_t}\left[\pi_{z_1}b_{z_1}(o_1)\prod_{\tau=2}^{t}a_{z_{\tau-1}z_{\tau}}b_{z_{\tau}}(o_{\tau})\right],
]

其中 \(\pi_{z_1}\) 为初始攻击原型分布。为避免直接枚举全部路径带来的高计算量，本文采用 **Viterbi / 最大后验路径识别算法** 对当前节奏进行在线估计。

定义

[
\delta_t(k)=\max_{z_1,\dots,z_{t-1}}P(z_1,\dots,z_{t-1},z_t=k,o_1,\dots,o_t),
]

表示在时刻 \(t\) 以状态 \(k\) 结尾的最优路径概率，同时定义回溯指针

[
\psi_t(k)=\arg\max_j\big[\delta_{t-1}(j)a_{jk}\big].
]

则 Viterbi 递推为：

**初始化：**
[
\delta_1(k)=\pi_k b_k(o_1),\qquad \psi_1(k)=0.
]

**递推：**
[
\delta_t(k)=b_k(o_t)\max_j\big[\delta_{t-1}(j)a_{jk}\big],\qquad t=2,\dots,T.
]

**终止与回溯：**
[
z_T^*=\arg\max_k \delta_T(k),\qquad z_t^*=\psi_{t+1}(z_{t+1}^*),\ t=T-1,\dots,1.
]

最终取

[
\hat z_t=z_t^*,
]

作为当前时刻最可能的对手攻击原型。为增强实时性，本文在比赛过程中采用滑动时间窗机制，在每个决策时刻仅使用最近 \(L\) 次观测动作构成局部序列 \(O_{t-L+1:t}\)，再执行一次 Viterbi 识别。这样既能保持对当前节奏的敏感性，又能避免过久远动作对状态判断产生干扰。

该识别层的引入，使问题3不再停留于“已知对手节奏，求我方最优动作”，而转化为“**先识别对手节奏，再进行最优响应**”。这一步也把问题2中“按攻击类别选防守动作”的静态映射，升级成了“动作观测—节奏识别—状态更新—最优响应”的动态闭环。

## 五、单步收益函数

### 1. 攻击动作收益

对攻击动作 (i)，用问题1已有结果定义：

[
R^{\text{atk}}_i
================

\alpha_1 A_i+\alpha_2 q_i-\alpha_3 e_i-\alpha_4(1-S_i)-\alpha_5 T_i.
]

含义分别为：

* (A_i)：打击效果；
* (q_i)：后续连招衔接性；
* (e_i)：失手后的暴露风险；
* (1-S_i)：动作引发的失稳风险；
* (T_i)：动作耗时成本。

### 2. 防守动作收益

对于防守动作 (j)，依问题2中攻击—防守匹配模型，防守收益定义为：

[
R^{\text{def}}_{k,j}
====================

\beta_1 P_{kj}+\beta_2 Q_{kj}+\beta_3 C_{kj},
]

其中：

* (P_{kj})：主防匹配得分；
* (Q_{kj})：稳定保障得分；
* (C_{kj})：反制机会得分。

这个式子本质上衡量：
面对攻击原型 (k) 时，动作 (j) 能否**防得住、站得稳、守后能反打**。

### 3. 统一即时回报

于是，在由节奏识别层更新得到状态 \((s_t,\hat z_t)\) 并选择动作 \(a_t\) 后，即时收益写成

[
r(s_t,a_t)=
\omega_1 \Pi_t^{\text{score}}
+\omega_2 \Pi_t^{\text{control}}
+\omega_3 \Pi_t^{\text{counter}}
-\omega_4 \Pi_t^{\text{loss}}
-\omega_5 \Pi_t^{\text{risk}}
-\omega_6 \Pi_t^{\text{time}}.
]

其中：

* (\Pi_t^{\text{score}})：本次动作造成有效攻击的期望收益；
* (\Pi_t^{\text{control}})：压迫、逼退、破坏平衡带来的场面控制收益；
* (\Pi_t^{\text{counter}})：防后反制的收益；
* (\Pi_t^{\text{loss}})：被对手命中的损失；
* (\Pi_t^{\text{risk}})：自身失衡风险；
* (\Pi_t^{\text{time}})：动作耗时带来的节奏损耗。

## 六、核心优化模型：半马尔可夫决策过程（SMDP）

因为不同动作持续时间不同，所以决策方程写为

[
V(s)=\max_{a\in \mathcal A(s)}
\left{
r(s,a)+
\gamma^{\Delta t(a)}
\sum_{s'}P(s'\mid s,a)V(s')
\right},
]

其中：

* (\Delta t(a)) 为动作 (a) 的持续时间；
* (P(s'\mid s,a)) 为状态转移概率；
* (\gamma\in(0,1)) 为折扣因子；
* (V(s)) 为状态价值函数。

这就是问题3的主模型。

### 为什么这里必须用 SMDP？

因为题目中的动作时间差异明显：直拳、勾拳属于短时动作，前踢、侧踢、冲撞属于中等动作，回旋踢、五连踢、倒地反击属于长时动作。若仍用普通 MDP，把所有动作视为相同时长，会高估长动作的连续使用价值。

## 七、动作序列搜索：识别驱动的 MCTS + 时间扩展网络

仅有 (V(s)) 还不够，题目还要求“先后采取哪些攻击动作与防守动作”。

因此，再加两层算法。

### 1. 在线搜索：MCTS

在当前识别状态 \((s,\hat z_t)\) 下，用蒙特卡洛树搜索生成未来若干步动作链。识别结果 \(\hat z_t\) 一方面决定我方优先调用的问题2防守动作池，另一方面约束候选攻击动作的风险评估。选择公式采用 UCT：

[
a^*=\arg\max_a
\left[
Q(s,a)+c\sqrt{\frac{\ln N(s)}{N(s,a)}}
\right],
]

其中：

* (Q(s,a))：动作 (a) 的平均回报；
* (N(s))：节点访问次数；
* (N(s,a))：边访问次数；
* (c)：探索系数。

这样做的好处是：
价值迭代告诉我们“哪类动作长期更好”，MCTS 则告诉我们“当前局面下接下来三步、五步怎么走更好”。

### 2. 可解释序列提取：时间扩展网络 DP

为了给论文输出一个清晰的动作链，再在高价值动作子集上建立时间扩展网络。设节点为动作，边权为两动作间可衔接收益，则序列优化写为：

[
F_{n+1}(v)=\max_{u\in \mathcal N(v)}
\left{
F_n(u)+\phi(v)+\psi(u,v)
\right},
]

其中：

* (\phi(v))：动作 (v) 的单步收益；
* (\psi(u,v))：从动作 (u) 过渡到动作 (v) 的衔接收益；
* (\mathcal N(v))：可衔接前驱动作集合。

这里的衔接收益 (\psi(u,v)) 主要由问题1中的衔接性 (q_i) 和问题2中的“防后反制能力”共同决定。问题1已指出：直拳、勾拳、组合拳、膝撞衔接性较高，而侧踢、回旋踢、五连踢、倒地反击衔接性较低。

## 八、算法流程伪代码

为使“节奏识别—动作决策”闭环过程更加清晰，可将问题3的在线决策流程概括为如下伪代码：

```text
算法 1  单场竞技赛的识别—决策联合优化流程
输入：初始状态 s_0，攻击原型转移矩阵 A，动作发射矩阵 B，初始分布 π，
      攻击动作集 A_atk，防守动作集 A_def，时间窗长度 L，规划深度 H
输出：比赛全过程最优动作序列 Π*

1: 初始化 t ← 0，观测窗口 O ← ∅，策略序列 Π* ← []
2: while 比赛未结束 do
3:     观测对手新动作 o_t，并更新窗口 O ← {o_{t-L+1},...,o_t}
4:     使用 Viterbi 算法在窗口 O 上计算最大后验路径 Z*
5:     取当前最可能攻击原型 \hat{z}_t ← Z* 的末状态
6:     根据 \hat{z}_t 调用问题2中的候选防守动作集 A_def(\hat{z}_t)
7:     结合距离、平衡与剩余时间状态，构造候选动作集
8:           A_t ← A_atk ∪ A_def(\hat{z}_t)
9:     在状态 (s_t, \hat{z}_t) 下执行 SMDP 价值评估与 MCTS 搜索
10:    得到当前最优动作 a_t*
11:    执行动作 a_t*，获得即时收益 r(s_t,a_t*)
12:    按状态转移概率更新比赛状态 s_{t+1}
13:    将 a_t* 加入策略序列 Π*
14:    t ← t+1
15: end while
16: 返回 Π*
```

## 九、算法收敛性说明

这部分很适合写进论文，回应题目对“收敛性与最优解有效性”的要求。

1. **Viterbi / 最大后验路径识别**：
   在给定攻击原型转移矩阵与动作发射矩阵后，Viterbi 递推满足最优子结构，可在有限步内稳定求得最大后验路径，不会因单步贪心而破坏全局最优识别。

2. **SMDP 价值迭代**：
   当 (\gamma\in(0,1)) 时，Bellman 算子为压缩映射，因此价值迭代收敛到唯一最优价值函数。

3. **MCTS（UCT）**：
   随着仿真次数增加，UCT 对最优动作的估计是一致的，访问次数趋于无穷时会收敛到最优决策。

4. **时间扩展网络 DP**：
   属于有限阶段动态规划，在给定时域和动作集下可精确求得最优路径，不存在局部最优陷阱。
